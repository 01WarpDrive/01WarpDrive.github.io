hexo+typora图片插入方案

前言

Hexo 是目前比较流行的个人博客网站构建工具，文章基于 Markdown 撰写。

然而当前 Hexo 官方提供的图片插入方式往往与流行的 Markdown 编辑器不匹配。插入方式略显繁琐。

笔者使用 Typora 作为笔记软件，尝试寻求一种快捷的图片插入方式。

在hexo + typora 图片插入解决办法 - Yumine - 博客园 (cnblogs.com)中，这位博主介绍了两种方案。一种是修改 Typora 默认图片保存地址为 Hexo 的图片目录。另一种是图床方案。

两者都存在一些缺陷：

  • 前者在修改后，对于那些不想提交到博客的笔记，图片也会被保存到 Hexo 图片目录，从而被提交到 Github 仓库。不仅耗费空间，而且造成信息泄漏。
  • Typora 默认的图片插入方式形如 ./.../xx.png和.../xx.png 都无法被 Hexo 正确识别。只有/img/xx.png能被 Hexo 识别，而这样的形式必须手动修改，过于繁琐。
  • 图床方案依赖图床平台，不稳定，插入 Typora 也不方便。

此外，在md img render part improvement of hexo-render-marked · Issue #216 · hexojs/hexo-renderer-marked (github.com)中，使用者们也讨论出了一种方案。

该方案首先要求设置 Hexo 中post_asset_folder: true，从而 Hexo 会为每一篇文章创建一个专门的文件夹存储图片。

笔者比较排斥这个方案，当发布的文章数量多起来后，Hexo 文件夹中会看起来十分臃肿，不利于笔者维护博客文章。

因此笔者提出了一种新的图片插入方案：

  • 无须启用 post_asset_folder: true
  • 图片保存在本地，且非 Hexo 文章的 md 文件图片不会被上传
  • 同时满足 Typora 和 博客网站的 Markdown 图片显示
